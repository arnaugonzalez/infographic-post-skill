    - Return dict with 5 keys: "coverage", "docstrings_missing", "oss_files", "flake8_issues", "complexity_hotspots"
    - Call _run_coverage(root) for coverage
    - Call _collect_docstring_gaps(root / "scripts") for docstrings_missing
    - Call _check_oss_files(root) for oss_files
    - Call _run_flake8(root / "scripts") for flake8_issues
    - Call _collect_hotspots(root / "scripts") for complexity_hotspots

    **_render(audit: dict) -> str:**
    Build Markdown string with these exact section headers (in order):
    ```
    # Quality Audit Report

    *Generated by oss_audit.py*

    ## Test Coverage

    | Module | Coverage |
    |--------|----------|
    | {path} | {pct:.1f}% |

    (If empty: "No coverage data available. Install `requirements-audit.txt` and run again.")

    ## Docstring Coverage

    Functions/classes missing docstrings:

    - {each entry from docstrings_missing}

    (If empty: "All functions and classes have docstrings.")

    ## OSS Baseline Files

    | File | Present |
    |------|---------|
    | {name} | Yes / **MISSING** |

    ## Code Quality (Flake8)

    | File | Line | Code | Message |
    |------|------|------|---------|
    | {file} | {line} | {code} | {message} |

    (If empty: "No logical errors found." or "Flake8 not available. Install `requirements-audit.txt`.")

    ## Complexity Hotspots

    Functions with >= {COMPLEXITY_THRESHOLD} branch nodes:

    | File | Function | Line | Branches |
    |------|----------|------|----------|
    | {file} | {name} | {line} | {branches} |

    (If empty: "No high-complexity functions found.")
    ```
